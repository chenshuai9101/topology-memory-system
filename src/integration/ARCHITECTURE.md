# 拓扑记忆系统集成架构

## 系统架构概述

拓扑记忆上下文管理器采用模块化、可扩展的架构设计，通过系统集成模块将所有组件统一管理。系统遵循依赖注入原则，支持多环境配置和全面的健康监控。

## 核心组件详解

### 1. 配置管理器 (ConfigManager)

#### 功能特性
- **多环境支持**: 开发、测试、预发布、生产环境
- **配置继承**: 默认配置 → 环境配置 → 环境变量
- **类型安全**: 使用数据类定义配置结构
- **动态验证**: 启动时验证配置有效性

#### 配置层次结构
```
环境变量 (最高优先级)
    ↓
环境配置文件 (development.yaml, production.yaml)
    ↓
默认配置文件 (default.yaml)
    ↓
代码默认值 (最低优先级)
```

#### 配置类别
- **数据库配置**: PostgreSQL连接参数
- **Redis配置**: 缓存和会话管理
- **向量搜索配置**: Qdrant/Weaviate连接
- **ML配置**: 嵌入模型和参数
- **API配置**: FastAPI服务器设置
- **日志配置**: 日志级别和输出
- **监控配置**: 健康检查和指标

### 2. 服务容器 (ServiceContainer)

#### 设计模式
- **依赖注入**: 自动解析服务依赖
- **控制反转**: 服务生命周期由容器管理
- **服务定位器**: 统一的服务获取接口

#### 服务生命周期
- **单例 (Singleton)**: 全局唯一实例
- **瞬时 (Transient)**: 每次请求创建新实例
- **作用域 (Scoped)**: 作用域内单例（如请求作用域）

#### 服务注册
```python
# 单例服务
container.register_singleton(DatabaseService)

# 瞬时服务  
container.register_transient(RequestProcessor)

# 作用域服务
container.register_scoped(UserSession)

# 工厂函数
container.register(Logger, factory=create_logger)
```

### 3. 系统集成器 (SystemIntegrator)

#### 启动流程
1. **初始化配置**: 加载和验证配置
2. **注册服务**: 初始化服务容器
3. **连接数据库**: 建立数据库连接池
4. **连接Redis**: 建立缓存连接
5. **连接向量搜索**: 初始化向量数据库
6. **初始化ML服务**: 加载嵌入模型
7. **启动API服务器**: 启动FastAPI应用
8. **启动健康监控**: 开始定期检查

#### 状态管理
- **集成状态**: 跟踪系统初始化进度
- **错误收集**: 集中处理启动错误
- **性能指标**: 记录各阶段启动时间
- **健康状态**: 实时监控系统健康

### 4. 健康检查器 (HealthChecker)

#### 检查类型
- **配置检查**: 验证配置有效性
- **连接检查**: 数据库、Redis、向量搜索连接
- **服务检查**: API端点、ML模型可用性
- **资源检查**: 内存、磁盘、CPU使用率
- **业务检查**: 自定义业务逻辑检查

#### 严重性分级
- **关键 (Critical)**: 系统无法运行
- **高 (High)**: 主要功能受影响
- **中 (Medium)**: 次要功能受影响
- **低 (Low)**: 不影响功能，需要关注

#### 监控机制
- **定期检查**: 可配置的检查间隔
- **超时处理**: 防止检查阻塞系统
- **结果缓存**: 避免频繁重复检查
- **告警通知**: 严重问题即时通知

### 5. 应用启动器 (ApplicationStarter)

#### 启动阶段
```
NOT_STARTED → INITIALIZING → CONFIG_LOADED → SERVICES_INITIALIZED
    ↓
DATABASE_MIGRATED → API_SERVER_STARTING → READY
    ↓
SHUTTING_DOWN → STOPPED
```

#### 特性
- **分阶段启动**: 清晰的启动进度反馈
- **优雅关闭**: 处理SIGINT/SIGTERM信号
- **错误恢复**: 启动失败时的清理操作
- **性能监控**: 记录各阶段启动时间

## 数据流设计

### 启动数据流
```
用户命令 → 启动脚本 → 应用启动器 → 系统集成器
    ↓
配置管理器 → 服务容器 → 各组件初始化
    ↓
健康检查器 → 监控线程 → API服务器
```

### 运行时数据流
```
API请求 → 服务容器 → 业务服务 → 数据访问
    ↓
健康检查 → 状态报告 → 监控系统
    ↓
配置更新 → 热重载 → 服务更新
```

## 扩展点设计

### 1. 配置扩展
```python
# 添加新配置类别
@dataclass
class NewServiceConfig:
    endpoint: str = "https://api.example.com"
    timeout: int = 30
    retries: int = 3

# 在ConfigManager中集成
class ConfigManager:
    def _init_new_service_config(self) -> NewServiceConfig:
        config = self._config.get("new_service", {})
        return NewServiceConfig(**config)
```

### 2. 服务扩展
```python
# 创建新服务
class NewService:
    def __init__(self, config: ConfigManager, logger: logging.Logger):
        self.config = config
        self.logger = logger
    
    def do_work(self):
        # 业务逻辑
        pass

# 注册服务
container.register_singleton(NewService)

# 自动依赖注入
other_service = container.create_with_dependencies(OtherService)
```

### 3. 健康检查扩展
```python
# 创建自定义检查
def check_custom_service():
    start_time = time.time()
    try:
        # 检查逻辑
        return HealthCheckResult(
            name="custom_service",
            status=HealthStatus.HEALTHY,
            severity=CheckSeverity.MEDIUM,
            message="Custom service is working",
            duration=time.time() - start_time,
            timestamp=time.time()
        )
    except Exception as e:
        # 错误处理
        pass

# 注册检查
checker.register_check(
    name="custom_service",
    check_func=check_custom_service,
    interval=60,
    severity=CheckSeverity.MEDIUM
)
```

## 性能优化

### 启动优化
- **并行初始化**: 独立组件并行初始化
- **懒加载**: 按需加载服务和资源
- **缓存机制**: 重复使用的实例缓存
- **连接池**: 数据库和Redis连接复用

### 运行时优化
- **服务复用**: 单例服务减少创建开销
- **连接复用**: 持久化连接减少握手
- **批量操作**: 数据库和向量搜索批量处理
- **异步处理**: I/O密集型操作异步化

### 监控优化
- **检查合并**: 相关检查合并执行
- **结果缓存**: 检查结果短期缓存
- **增量检查**: 只检查变化的部分
- **智能间隔**: 根据重要性调整检查频率

## 故障恢复

### 启动失败恢复
- **配置回退**: 无效配置使用默认值
- **服务降级**: 非关键服务失败继续启动
- **错误报告**: 详细的启动错误信息
- **清理操作**: 失败时的资源清理

### 运行时故障恢复
- **健康检查**: 实时检测服务故障
- **自动重启**: 关键服务自动重启
- **故障转移**: 备用服务接管
- **状态同步**: 恢复后状态同步

### 数据一致性
- **事务管理**: 数据库操作事务性
- **幂等操作**: 重复操作结果一致
- **状态检查**: 操作前后状态验证
- **恢复日志**: 故障恢复操作记录
